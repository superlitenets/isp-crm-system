✅ Root Cause Confirmed

The error:

wifi_bridge_config:34:13
Invalid parameter path


Line 34:

var wan = declare("InternetGatewayDevice.WANDevice.1.WANConnectionDevice.", null, {path: 1});


This is invalid in GenieACS.

Why?

Because:

WANConnectionDevice.


is not a parameter path
and not a valid instance pattern.

GenieACS requires * when working with instances.

Without *, it tries to parse it as a parameter name and fails.

That’s why you get:

Invalid parameter path

🧠 Why This Is Subtle

This syntax sometimes appears in older examples online, but:

It only works in certain contexts

It fails depending on device tree state

It is not safe for dynamic provisioning

Your current firmware + CWMP engine is strict.

✅ Correct Instance Handling in GenieACS

To work with instance objects, you must use:

*.*


pattern.

Correct way:

Step 1 — Count existing WANConnectionDevice instances
var wans = declare("InternetGatewayDevice.WANDevice.1.WANConnectionDevice.*", {path: 1});

Step 2 — Create new instance
declare("InternetGatewayDevice.WANDevice.1.WANConnectionDevice.*", null, {path: wans.length + 1});

Step 3 — Find highest instance index
var wansAfter = declare("InternetGatewayDevice.WANDevice.1.WANConnectionDevice.*", {path: 1});

var maxIndex = 0;

for (var i in wansAfter) {
  var index = parseInt(wansAfter[i].path.match(/WANConnectionDevice\.(\d+)/)[1]);
  if (index > maxIndex) maxIndex = index;
}

var newWanPath = "InternetGatewayDevice.WANDevice.1.WANConnectionDevice." + maxIndex + ".";

Step 4 — Create WANIPConnection properly
var ipList = declare(newWanPath + "WANIPConnection.*", {path: 1});
declare(newWanPath + "WANIPConnection.*", null, {path: ipList.length + 1});

Step 5 — Resolve new IP path
var ipAfter = declare(newWanPath + "WANIPConnection.*", {path: 1});

var maxIp = 0;
for (var i in ipAfter) {
  var index = parseInt(ipAfter[i].path.match(/WANIPConnection\.(\d+)/)[1]);
  if (index > maxIp) maxIp = index;
}

var ipPath = newWanPath + "WANIPConnection." + maxIp + ".";
