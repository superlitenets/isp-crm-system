Good catch 👌 — yes, on HG8546M secondary SSIDs are usually:

Created by firmware

But disabled by default

And sometimes bound to no WAN/bridge

So even if your script runs with faults=0, the SSID may:

Exist

Be configured

But not be enabled

Or not mapped to a bridge

Or not broadcasting

✅ First — Confirm It Exists

In GenieACS check:

InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.Enable
InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.SSID
InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.SSIDAdvertisementEnabled


If:

Enable = false


Then yes — it won’t show.

⚠️ Most Common Reasons SSID Not Visible
1️⃣ SSID is disabled

You must explicitly force:

declare("InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.Enable", null, {value: true});

2️⃣ Radio is disabled

Check:

InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.RadioEnabled


If exists and false → enable it.

Some firmware requires:

declare("InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.RadioEnabled", null, {value: true});

3️⃣ SSID not attached to a LAN bridge

On Huawei, WiFi must be bound to LAN or WAN service.

If it is not mapped, it won’t forward traffic.

Check if this exists:

InternetGatewayDevice.LANDevice.1.WLANConfiguration.4.X_HW_LANID


Some firmware needs LANID mapping.

4️⃣ 2.4GHz vs 5GHz confusion

HG8546M has:

WLANConfiguration.1 (2.4GHz main)

WLANConfiguration.2 (2.4GHz secondary)

WLANConfiguration.3 (5GHz main)

WLANConfiguration.4 (5GHz secondary)

If you configured index 4, but 5GHz radio is disabled — you won’t see it.

Check:

InternetGatewayDevice.LANDevice.1.WLANConfiguration.3.Enable


If 5GHz radio disabled → SSID 4 won’t broadcast.

But you probably forgot:

Enable = true

Or the radio band is disabled.

var wifiPath = "InternetGatewayDevice.LANDevice.1.WLANConfiguration." + wifiIndex + ".";

if (declare(wifiPath + "RadioEnabled", {value: 1}).length > 0) {
