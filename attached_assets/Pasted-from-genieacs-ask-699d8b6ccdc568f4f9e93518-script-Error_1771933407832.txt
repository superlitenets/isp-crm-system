You are likely sending something like:

or

{
  "name": "instances",

Those are provision script functions, not valid task names in the REST API.

GenieACS REST API supports:

addObject

deleteObject

setParameterValues

getParameterValues

reboot

It does NOT support:

refresh

instances

declare

commit

Those are only valid inside provision scripts.

🎯 The Correct Way To Force Rediscovery From API

You cannot use:

"name": "refreshObject"

That is only valid inside a provision script.

From the REST API you must use:

✅ Correct Method

After AddObject:

POST /devices/{id}/tasks?connection_request
{
  "name": "getParameterNames",
  "parameterPath": "InternetGatewayDevice.WANDevice.1.WANConnectionDevice.",
  "nextLevel": true
}

That forces GenieACS to rediscover children under that path.

NOT refreshObject.

NOT refresh.

NOT instances.
