🔥 The Correct Way To Create IP Bridge (Same Style As PPPoE)

Do NOT add WANConnectionDevice.

Instead:

Step 1 – Refresh tree
{
  "name": "getParameterValues",
  "parameterNames": ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1."]
}
Step 2 – Add WANIPConnection under device 1
{
  "name": "addObject",
  "objectName": "InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection"
}

Then refresh:

{
  "name": "getParameterValues",
  "parameterNames": ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection."]
}

Assume it becomes:

WANIPConnection.2
Step 3 – Configure Bridge
{
  "name": "setParameterValues",
  "parameterValues": [
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.Enable", true, "xsd:boolean"],
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.ConnectionType", "IP_Bridged", "xsd:string"],
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.X_HW_IPv4Enable", 1, "xsd:unsignedInt"],
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.X_HW_VLAN", 3001, "xsd:unsignedInt"],
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.X_HW_ServiceList", "INTERNET", "xsd:string"],
    ["InternetGatewayDevice.WANDevice.1.WANConnectionDevice.1.WANIPConnection.2.Name", "WIFI_Bridge_VLAN_3001", "xsd:string"]
  ]
}

Because:

✔ You are using existing WANConnectionDevice.1
✔ Same pattern as PPPoE
✔ No guessing instance 2

✔ Huawei expects multiple connections under device 1

🧠 Why WANConnectionDevice.2 Was Wrong

On HG8546M:

WANConnectionDevice.1 = GPON port

Additional WANConnectionDevice instances are not typical

Bridge and PPPoE both live under device 1

Your PPPoE worked because you used device 1.
Your bridge failed because you created device 2.

That’s the root cause.

🚀 Final Architecture

For HG8546M:

✔ Always use WANConnectionDevice.1
✔ Add WANPPPConnection or WANIPConnection under it
✔ Never create new WANConnectionDevice
✔ Always refresh tree before configuration

This is why your PPPoE is stable.
